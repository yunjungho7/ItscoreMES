## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> API | Untrusted input (queries, body) enters the system. |
| API -> DB | SQL queries executed; results returned. |
| API -> XML Mappers | Local file system access for SQL templates. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Logging | mitigate | Filter PII and secrets from logs; use structured JSON. |
| T-01-02 | Tampering | SQL Injection | mitigate | Use SQLAlchemy parameterized queries (already partially implemented via XMLMapper). |
| T-01-03 | Information Disclosure | XML Mapper | mitigate | Use `defusedxml` to prevent XXE attacks reading local files. |
| T-01-04 | Denial of Service | DB Connections | mitigate | Use SQLAlchemy connection pooling to prevent connection exhaustion. |
</threat_model>
